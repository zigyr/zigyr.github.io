## 前置知识

### 重定向符号 > 与 > >

- **>**：覆盖写入，文件不存在则创建，存在则清空原有内容后写入
```shell
echo benben > a
```

- **>>**：追加写入，文件不存在则创建，存在则在末尾新增内容
```shell
echo benben >> a
```

- **> 文件名**：等价于 `touch`，快速创建空文件
```shell
> a
```


### 命令换行符 \

命令末尾添加**反斜杠 \**，实现单行命令多行拆分书写，拼接合并执行。

```shell
ca\
> t \
> fla\
> g.php
```


### ls -t 时间排序

- 放弃字母排序，**按修改时间倒序**展示，后创建文件优先置顶
- 限制：精准时间记录有限，不支持复制粘贴式批量建文件精准排序

```shell
>c
>a
>b

ls -t

-> a b c
```


### dir 命令特性

功能与 `ls` 基本一致，独有优势：
- 字母排序优先级靠前
- 默认按列紧凑输出，减少换行

```shell
ls
dir
```

- 重定向差异：
  - `ls > 文件名`：逐行单列写入
  - `dir > 文件名`：多列紧凑写入


### 通配符 * 作用

- **\*** 等效于 `$(dir *)`，自动展开当前目录所有文件
- 规则：若首个文件名为系统命令，直接执行该命令，后续文件名作为命令参数

```shell
$(dir *)
*
```


### rev 反转命令

- 作用：逐行反转文件内容并控制台回显
- 基础用法：
```shell
rev flag
```


## Q&A
### `$()`叫什么

名称：**命令替换（Command Substitution）**
- 执行括号内命令
- 截取命令标准输出
- 将结果嵌入当前命令行二次执行


### dir 与 dir * 区别

- dir
	仅默认罗列常规目录内容，特殊文件名、异常字符文件存在展示不全问题。

- dir *
	**\*** 为 Bash 通配符，强制全量展开所有文件
	覆盖场景：正常文件名、特殊符号名、控制字符、隐藏文件、畸形命名文件
