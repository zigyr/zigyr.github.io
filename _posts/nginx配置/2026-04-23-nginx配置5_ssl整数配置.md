## 前言

`ssl证书配置`其实就是`http`转向`https`的过程

由于ssl证书的配置需要一定的财力

目前还无法一一实践

以下侧重演示`在阿里云申请的nginx的ssl证书`的配置

## HOW

### fri

依然结合我们多站点配置的思想

在`/etc/nginx/conf.d`中，创建一个`www.wulaoban.top.conf`的站点配置文件

进行如下配置

```nginx
server {
    listen 443 ssl;
    server_name www.wulaoban.top;

    #ssl_certificate cert/<cert-file-name>.pem;
    ssl_certificate /opt/cert/9683539_wulaoban.top.pem;     #填写证书文件名称
    #ssl_certificate_key cert/<cert-file-name>.key;
    ssl_certificate_key /opt/cert/9683539_wulaoban.top.key; #填写证书私钥文件名称  

    ssl_session_timeout 5m;

    #表示使用的加密套件的类型
    ssl_ciphers ECDHE-RSA-AES128-GCMSHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;

    #表示使用的TLS协议的类型，默认走TLSv1.3
    ssl_protocols TLSv1.1 TLSv1.2 TLSv1.3; 
    ssl_prefer_server_ciphers on;

    location / {
        root /web/www.wulaoban.top;
        index index.html index.htm;
    }
}
```


### sec

注意修改主机上的`hosts`文件

使得`www.wulaoban.top`这个域名执行我们部署nginx的服务器

其次，在后续访问的过程中注意url前缀是`https`
